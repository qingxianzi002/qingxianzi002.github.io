Spring

一、前言

  • Spring Framework 是最早，且是底层框架
  • Spring Boot 提高开发速度，降低成本
  • Spring Cloud 分布式开发相关技术

二、Spring Framework框架构成

  • Core Container 核心容器 ——–> 装对象
  • AOP 面向切面编程 ———-> 依赖于Core Container（核心容器）
  • Aspects 实现 AOP
  • Data Access 数据访问
  • Data Integration 数据集成
  • Transactions 事务
  • Test 单元测试与集成测试

三、Spring IOC

  • 含义为控制反转

  • /*业务层实现*/
public class BookServiceImpl implements BookService{
    private BookDao bookDao;
    public void save(){
        bookDao.save();
    }
}

/*数据层实现*/
public class BookDaoImpl implements BookDao{
    public void save(){
        System.out.println("book dao save ...");
    }
}

/*
以上在业务层中调用BookDao的对象，如果有多个BookDao的实现类
--平常方法：直接在业务层的bookDao后面new几个对象（这种方法耦合度过高）
--降低耦合度的方法：在另外创建IoC容器，里面存放service、dao层等对象
--由此推出控制反转思想是将对象创建控制权由程序转移到外部，而IoC容器充当外部角色，创建和管理的对象充当Bean角色
*/

  • IoC容器中service层和dao层是依赖关系（即两个Bean之间是依赖关系）———-> 依赖注入（DI）
